## Understanding Changelog Format

Changelogs follow the [Keep a Changelog](https://keepachangelog.com/) format with semantic versioning:

<Tabs>
  <Tab title="Major Changes">
    Breaking changes that require migration steps.

    **Example:**
    ```markdown
    ## 1.0.0

    ### Major Changes

    - refactor(core): remove old container API

    BREAKING CHANGE: the previous `createContainer` method is replaced with a new interface
    ```
  </Tab>
  <Tab title="Minor Changes">
    New features that are backward compatible.

    **Example:**
    ```markdown
    ## 0.7.0

    ### Minor Changes

    - feat: add new caching strategies
    - feat: support custom providers
    ```
  </Tab>
  <Tab title="Patch Changes">
    Bug fixes and minor improvements.

    **Example:**
    ```markdown
    ## 0.6.1

    ### Patch Changes

    - fix: resolve circular dependency issue
    - chore: update dependencies
    ```
  </Tab>
</Tabs>

## Staying Updated

<Steps>
  <Step title="Watch the repository">
    [Watch the repository](https://github.com/resolid/framework) on GitHub to receive notifications about new releases.
  </Step>
  <Step title="Check release notes">
    Review the [releases page](https://github.com/resolid/framework/releases) for detailed information about each version.
  </Step>
  <Step title="Follow migration guides">
    For breaking changes, follow the migration guides provided in the release notes.
  </Step>
  <Step title="Update dependencies">
    Keep your dependencies up to date using your package manager:
    ```shell
    pnpm update @resolid/*
    ```
  </Step>
</Steps>

<Note>
  The changelog is automatically generated from commit messages following the [Conventional Commits](https://www.conventionalcommits.org/) specification.
</Note>
